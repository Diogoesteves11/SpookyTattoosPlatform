--------------------------------------------------------
PARTE 1: TESTES FUNCIONAIS (CRUD E REGRAS DE NEGÓCIO)
--------------------------------------------------------
Instruções para correr a bateria exaustiva de 20 testes 
(validação de Views, Triggers e Constraints).

OPÇÃO A: Via Terminal (Rápido e Automático)
# Corre este comando na raiz do projeto (onde está o docker-compose.yml):
docker exec -i spookytattoos_db psql -U admin -d spookytattoos_db < database/tests/01_comprehensive_tests.sql

* NOTA SOBRE ERROS NO TERMINAL: 
É normal e esperado que o terminal devolva mensagens de "ERROR" nos testes 3, 8, 18 e 19. 
Isso prova que as restrições de segurança (emails únicos, chaves estrangeiras, limites de datas) estão a funcionar.

OPÇÃO B: Via pgAdmin (Visual e Interativo)
# Ideal para ver as Views formatadas em tabelas:
1. Abre o browser e vai a http://localhost:5050
2. Faz login com as credenciais pgAdmin definidas no teu .env
3. No menu lateral esquerdo, expande: Servers -> (Teu Servidor) -> Databases -> spookytattoos_db
4. Clica com o botão direito em "spookytattoos_db" e escolhe "Query Tool"
5. Clica no ícone de abrir ficheiro (pasta) e seleciona o teu "01_comprehensive_tests.sql" (ou copia e cola o conteúdo do ficheiro para a janela)
6. Clica no botão "Execute/Refresh" (o ícone de Play) para correr a bateria completa.


--------------------------------------------------------
PARTE 2: TESTES DE CARGA E PERFORMANCE (PGBENCH)
--------------------------------------------------------
Instruções para simular 50 utilizadores simultâneos a 
consumir a API e testar os limites do servidor.

# PASSO 1: Entrar no terminal do contentor PostgreSQL
docker exec -it spookytattoos_db bash

# PASSO 2: Criar o script temporário de stress
cat << 'EOF' > /tmp/stress_test.sql
SELECT * FROM vw_agenda_completa WHERE scheduled_date >= NOW();
SELECT * FROM vw_catalogo_publico LIMIT 20;
EOF

# PASSO 3: Executar o bombardeamento à Base de Dados
# Parâmetros: 50 clientes (-c), 4 threads (-j), durante 30 segundos (-T)
pgbench -U admin -d spookytattoos_db -f /tmp/stress_test.sql -c 50 -j 4 -T 30

# PASSO 4: Sair do contentor após ler os resultados (tps e latency)
exit
